# 🍕 Pizza Sales End-to-End Business Intelligence Project

## 📊 Project Overview
This project builds a comprehensive, enterprise-grade data analysis solution for a pizza restaurant chain. It spans database design, writing advanced SQL extraction queries, and engineering a 3-page interactive Power BI dashboard tracking revenue, product health, and kitchen operational kinetics.

### 🗂️ Dashboards Included:
1. **Sales Performance:** High-level executive overview of revenue metrics ($817.86K total revenue), total orders, and average order valuations.
2. **Product Performance (Best/Worst Sellers):** Granular visual parsing pinpointing highest and lowest grossing pizza categories, sizes, and quantities.
3. **Operational Insights:** Heavy data mapping highlighting peak hourly transaction rushes (lunch vs. dinner curves) and dynamic cross-sell matrix heatmaps.

---

## 🛠️ Tech Stack & Tools Used
* **Database Engine:** Microsoft SQL Server (SSMS)
* **Data Processing Logic:** SQL Queries (Joins, Aggregations, Date/Time intelligence)
* **Visualization Layer:** Power BI Desktop
* **Dynamic Navigation:** Page Navigator button controls, conditional cell shading heatmaps, custom drop-shadow containers

---

## 📊 Key Insights Extracted
* **Peak Rush Hours:** Peak order traffic forms two distinct operational peaks: lunch hours (12:00 PM - 1:00 PM) and a larger dinner surge (5:00 PM - 7:00 PM).
* **Consumer Dynamics:** The overwhelming majority of transactions consist of 1 or 2 pizzas per order, implying targeted meal sizing over massive bulk event catering.
* **Top Revenue Pillars:** Large Classic category pizzas stand out as a foundational revenue generator, visible via matrix color-gradient tracking.
